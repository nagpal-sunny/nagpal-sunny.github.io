Design Pattern Series: Template Pattern (Behaviour Pattern)

Posted Jun 24, 2023 Updated Sep 10, 2023
By
4 min read

Outline

Template - A structure/pattern to be followed for an algorithm/process.


The Template pattern declares/defines a structure for a process/algorithm in a superclass and the subclass can override specific steps of an algorithm to provide desired behaviour without changing the execution flow of the process. The method/function that decides the execution sequence of a process/algorithm is called the Template method.

Problem Statement

While writing code sometimes you might get into a situation where a standard set of steps to be performed to achieve the functionality. The easy option is to call these steps one by one at the required place. This way the desired functionality will be achieved however, the process is not standardised and not reusable.

For example, you are working on a news reader application. To display the news in the application a standard set of steps needs to be followed:

  1. Fetch the news from the source.
  2. parse the news feed.
  3. Categories the news (Entertainment, Top News etc.).
  4. Filter the duplicate news.
  5. Sort the Categories & News based on Timestamp.

Letâ€™s suppose you have written a code to support XML type of response initially. After 6 months of development, a new news feed source needs to be supported which has a JSON-type response. In this case, most of the steps will be the same only the parsing step is changed.

A straightforward approach will handle all the requests and responses in the common class. However, the common class will have a check based on response and request type. Another approach will be a Template method for getting the Feeds from the source.


Metaphor

Template patterns can be related to a car assembly. A Car assembly has a sequence of steps in the following order designing, Fit Car Body, Fit car chassis, Fit Car Engine, Fit Car Door.

Each step, such as designing, fitting a chassis, and fitting a car engine can be slightly changed based on the type of car but the sequence of steps will be the same.

Implementation

Steps for the process/algorithm declared in an interface and the template method defined in an abstract class or interface which defined the sequence of execution of steps.

Applicability

  1. Use the template pattern when you are implementing an algorithm which has a sequence of steps to follow. In future, if you need a variant of the algorithm with a step change then a derived class can be used and required changes to be done in the derived class. Example: Compiler designing.

  2. When you need a variant in algorithm/process based on data type or extension in that case template pattern can be used. For example parsing (XML/JSON/txt), upload data (image/Xml/txt). So most of the steps will be the same only the handling of data will be changed.

  3. A Process with multiple variants like authentication can be implemented with the Template method. There might be steps before and after authentication that will not affected by the authentication method (PIN/ FACEId/ Touch ID).

Trade-offs

** Reusability vs Flexibility **

Template patterns support code reusability by moving common code in the base class. On the other hand, the flexibility to add custom steps in specific variants is not possible with the template method. For example, In the above case, a new car requires the solar panel to be set up on the roof to support an electric battery which is not possible in the current template. There is no step in the process to add a Panel on the rooftop.
