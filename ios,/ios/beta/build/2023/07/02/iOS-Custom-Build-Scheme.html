Distribute iOS Beta Build (Scheme & Configurations)

Jul 2, 2023

Overview

It is a good practice to have multiple build configurations during the development phase of the application. Customers can ask for both debug and release configuration builds. Beta and Alpha testing is an important phase of the application life cycle. Beta and Alpha builds might have a different configuration from the release build. This blog explains one of the ways to handle this through an XCode scheme.

Problem Statement

Suppose you are working on an e-commerce application shoptime. In the development phase, multiple backend environments need to support (Dev, QA, UAT, Release). To differentiate between the application build (Development, QA, Beta builds etc) separate application icons are provided by the customer. These builds also have a unique bundle-id. The launch screens are also different for each build so that users can easily identify the build. Configuration can support multiple features however for this blog we will focus on the following:

  • Bundle Identifier
  • App Icon
  • Launch Screen
  • Backend URL

Implementation

XCode support multiple configurations in a single target. By default, XCode provides debug and release configuration. Let’s start step by step the process:

Step 1 Add a new configuration (Beta configuration)

Navigate to the project setting


Tap on the “+” button in the configuration section and duplicate the debug configuration(you can choose duplicate of release based on requirement).



Enter the name of the configuration (BetaDebug).

Step 2 Change product bundleid for BetaDebug

Move to Target-> Build Setting-> Search(Product Bundle Identifier)


Set the bundle identifier as per your requirement.

Step 2 Set separate application icon for BetaDebug

Add a new AppIcon asset in Assets with a specific name(AppIconBeta).


Add AppIconBeta in the BetaDebug configuration.


Finally!, a separate application icon is set for BetaDebug.
